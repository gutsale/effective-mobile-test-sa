# Задание 2. Проектирование API

## Описание
API используется для получения списка партнёрских магазинов.

---

## Пример запроса

```http
GET /api/v1/partner-stores
Host: api.petrushka-green.ru
Accept: application/json
Authorization: Bearer <access_token>
```

---

## Пример ответа

```json
{
  "data": [
    {
      "id": 1,
      "name": "METRO",
      "logo_url": "https://cdn.petrushka-green.ru/metro.png",
      "delivery": {
        "type": "time_interval",
        "text": "Ближайшая доставка сегодня 21:00–23:00",
        "start_time": "21:00",
        "end_time": "23:00"
      },
      "external_url": "https://metro.ru"
    },
    {
      "id": 2,
      "name": "ВкусВилл",
      "logo_url": "https://cdn.petrushka-green.ru/vkusvill.png",
      "delivery": {
        "type": "fast_delivery",
        "text": "Быстрая доставка от 20 до 60 минут",
        "min_minutes": 20,
        "max_minutes": 60
      },
      "external_url": "https://vkusvill.ru"
    }
  ],
  "meta": {
    "count": 2
  }
}
```

---

## Обоснование решения

Для получения списка партнёрских магазинов используется метод **GET**, так как операция является операцией чтения данных и не изменяет состояние системы.

Ответ API имеет следующую структуру:

- `data` — содержит массив партнёрских магазинов  
- `meta` — содержит дополнительную информацию о результате (в данном случае — общее количество элементов)

Каждый объект магазина включает:

- `id` — уникальный идентификатор магазина  
- `name` — название магазина  
- `logo_url` — ссылка на изображение логотипа  
- `external_url` — ссылка для перехода на внешний ресурс партнёра  

Блок `delivery` вынесен в отдельный объект внутри магазина для гибкости и расширяемости модели.

В зависимости от типа доставки используется поле `delivery.type`:

- `time_interval` — доставка в конкретный временной интервал (используются поля `start_time` и `end_time`)  
- `fast_delivery` — быстрая доставка с диапазоном времени (используются поля `min_minutes` и `max_minutes`)  

Поле `delivery.text` содержит готовую строку для отображения на клиенте, что позволяет упростить реализацию на стороне frontend и избежать дополнительного форматирования.

Поле `meta.count` позволяет клиенту быстро определить количество элементов в ответе и может использоваться для реализации пагинации в будущем.

Такая структура делает API понятным, расширяемым и удобным для интеграции с клиентскими приложениями.
