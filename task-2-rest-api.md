# Задание 2. Проектирование API

## Описание
API используется для получения списка партнёрских магазинов.

---

## Пример запроса

```http
GET /api/v1/partner-stores
Host: api.petrushka-green.ru
Accept: application/json
```

---

## Пример ответа

```json
{
  "data": [
    {
      "id": "a3f5c8d2-91b4-4e2a-9d1f-6c7e8a123456",
      "name": "METRO",
      "logo_url": "https://cdn.petrushka-green.ru/metro.png",
      "external_url": "https://metro.ru"
    },
    {
      "id": "b7e1d9f4-2c33-4a6f-8b1a-9f0c12345678",
      "name": "ВкусВилл",
      "logo_url": "https://cdn.petrushka-green.ru/vkusvill.png",
      "external_url": "https://vkusvill.ru"
    }
  ],
  "meta": {
    "count": 2
  }
}
```
## Коды ответов

- **200 OK** — успешный ответ, список партнёрских магазинов получен

- **400 Bad Request** — некорректный запрос

- **404 Not Found** — данные не найдены

- **500 Internal Server Error** — внутренняя ошибка сервера
---
## Формат ошибки

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Описание ошибки"
  }
}
```

---

## Обоснование решения

Для получения списка партнёрских магазинов используется метод **GET**, так как запрос предназначен для чтения данных.

В URL используется версионирование (`/api/v1/partner-stores`), что позволяет поддерживать и развивать API.

Ответ разделён на два блока:
- `data` — список партнёрских магазинов;
- `meta` — дополнительная информация о результате.

Такая структура делает ответ понятным и удобным для дальнейшего расширения.

Каждый объект магазина содержит:
- `id` — уникальный идентификатор магазина;
- `name` — название магазина;
- `logo_url` — ссылка на логотип;
- `external_url` — ссылка на внешний ресурс партнёра.

Поле `external_url` включено в ответ, так как по условию задачи при клике на плашку магазина должен выполняться переход на внешний ресурс.

Поле `meta.count` позволяет клиенту сразу получить количество элементов в ответе.

Также отдельно описаны коды ответов и формат ошибки, чтобы клиент мог корректно обрабатывать результат запроса.
